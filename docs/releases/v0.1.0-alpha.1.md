# Darjeeling v0.1.0-alpha.1

Darjeeling `v0.1.0-alpha.1` is the first public alpha preparation release.

Darjeeling is a runtime for LLM applications that turns well-bounded model
capabilities into local artifacts only when those artifacts stay inside checked
reliability boundaries. Hard or unfamiliar requests fall back to the main LLM
path.

## Install

The PyPI distribution name is `darjeeling-ai`. The Python import package remains
`darjeeling`, and the CLI command remains `darjeeling`.

From a checkout:

```bash
uv sync --extra dev
uv run darjeeling demo thin-target
```

After the package is published:

```bash
uvx --from darjeeling-ai --prerelease allow darjeeling demo thin-target
```

## Highlights

- Added `darjeeling demo thin-target`, a no-network toy demo that shows cold
  start through the simulated reference path, local artifact acceptance,
  fallback for unfamiliar requests, and a metrics report.
- Rewrote the README for first-time users.
- Renamed the distribution to `darjeeling-ai` to avoid the unrelated
  `darjeeling` PyPI package.
- Added CI, issue forms, changelog, contributing guide, security policy, and
  launch checklists.

## Alpha Status

This is an alpha runtime. It is ready for design validation and early demo
feedback.

Current limits:

- stores and queues are filesystem or in-memory implementations;
- the demo uses toy data and simulated reference responses;
- production hardening remains future work;
- no benchmark leadership or production-readiness claim is made.
