# Artifact Worker Module

## Purpose

The Artifact Worker module defines and enforces the single runtime protocol used by L1, L2, and L3 artifacts. It validates artifact packages, starts workers, sends requests, interprets `accept`/`abstain`, and converts failures into safe fallback events.

Core does not care whether an artifact is Rust code, Python code, a small model, an ONNX bundle, a local SLM wrapper, a table, or a ruleset. Core only requires the worker protocol, resource limits, and packaging boundary.

## Boundary

Inputs:

- `CandidateSubmission` from Agent Workspace.
- Artifact directories under `submissions/<candidate>/artifacts/l1|l2|l3`.
- `TargetDefinition` contract hash for package compatibility checks.
- Runtime worker protocol version supported by Core.
- Evaluation or runtime requests from Candidate Evaluation and Release Runtime.

Outputs:

- `ArtifactPackage` metadata to Candidate Evaluation and Release Runtime.
- `ArtifactCheckReport` to Candidate Evaluation.
- `LayerAttemptResult` to Candidate Evaluation, Release Runtime, and Trace.

An `ArtifactPackage` proves that a layer can be called. It does not imply that the layer is enabled in a given Candidate or Release. Candidate Evaluation and Release Runtime apply `RoutingSettings.enabled_layers` before calling artifacts.

This module must not:

- Call L4/reference.
- Call the next layer.
- Run compile-time scaffolding during serving.
- Accept artifacts that depend on validation/test data or registry credentials.

## Runtime Protocol

Worker request:

```json
{
  "request_id": "r-123",
  "input": {},
  "deadline_ms": 20
}
```

Worker response:

```json
{
  "decision": "accept",
  "output": {},
  "confidence": 0.997,
  "reason": "exact_pattern_42"
}
```

or:

```json
{
  "decision": "abstain",
  "confidence": 0.41,
  "reason": "outside_supported_region"
}
```

Core records `confidence` and `reason`, but does not impose a global threshold across all targets. The protocol field is named `reason` for compactness, but its durable meaning is `reason_code`: a short bounded identifier, not free-form explanation text.

## Data Types

### `ArtifactManifest`

Fields:

- `api_version: str`
- `layer: Literal["L1", "L2", "L3"]`
- `start_command: list[str]`
- `healthcheck_command: list[str] | None`
- `protocol: Literal["jsonl", "unix_socket"]`
- `timeout_ms: int`
- `memory_mb: int | None`
- `network: Literal["disabled"]`
- `contract_hash: str`
- `artifact_id: str | None`
- `allowed_reason_codes: list[str] | None`

### `ArtifactPackage`

Fields:

- `artifact_id: str`
- `layer: Literal["L1", "L2", "L3"]`
- `package_path: Path`
- `manifest: ArtifactManifest`
- `digest: str`
- `source_snapshot_digest: str`
- `build_provenance: dict`

### `WorkerRequest`

Fields:

- `request_id: str`
- `input: dict`
- `deadline_ms: int`

During private evaluation, `request_id` is a temporary opaque id generated by Core for that replay only. It must not be the source record id, snapshot record id, split index, trace id, or any stable id that lets an artifact recognize validation/test membership.

### `WorkerResponse`

Fields:

- `decision: Literal["accept", "abstain"]`
- `output: dict | None`
- `confidence: float | None`
- `reason: str | None`

`reason` must be a trace-safe reason code: bounded length, identifier-like, and not raw user data or free-form explanation text. If `ArtifactManifest.allowed_reason_codes` is present, non-null `reason` must be one of those codes.

### `LayerAttemptResult`

Fields:

- `layer: Literal["L1", "L2", "L3"]`
- `artifact_id: str`
- `decision: Literal["accept", "abstain", "error", "timeout", "invalid_output", "protocol_error"]`
- `output: dict | None`
- `confidence: float | None`
- `reason: str | None`
- `latency_ms: float`
- `error: str | None`

`LayerAttemptResult.reason` preserves the validated worker reason code. Runtime Trace stores it as `TraceLayerAttempt.reason_code`.

## Functions

### `read_artifact_manifest`

Input:

- `artifact_dir: Path`

Output:

- `ArtifactManifest`

Purpose:

- Read `artifact.yaml`.
- Validate required fields, layer value, protocol version, timeout, and start command.
- Validate `allowed_reason_codes` when present: bounded list, bounded code length, identifier-like strings, no free-form text.

Used by:

- `validate_artifact_package`

### `validate_artifact_package`

Input:

- `artifact_dir: Path`
- `expected_layer: Literal["L1", "L2", "L3"]`
- `contract_hash: str`
- `package_policy: PackagePolicy`

Output:

- `ArtifactCheckReport`

Purpose:

- Ensure the package is self-contained runtime content.
- Reject path escapes, symlinks outside package, forbidden files, forbidden network settings, missing manifests, and contract hash mismatches.
- Check that compile-time scaffolding and data are not included.

Used by:

- Candidate Evaluation preflight.
- Release creation.

### `freeze_artifact_package`

Input:

- `artifact_dir: Path`
- `manifest: ArtifactManifest`
- `artifact_store: ArtifactStore`

Output:

- `ArtifactPackage`

Purpose:

- Copy package bytes into immutable content-addressed storage.
- Compute digest and source snapshot digest.
- Return package metadata used by Candidate and Release.

Used by:

- Candidate Evaluation module.

### `start_worker`

Input:

- `package: ArtifactPackage`
- `worker_limits: WorkerLimits`

Output:

- `WorkerHandle`

Purpose:

- Start the artifact worker in a restricted serving/evaluation sandbox.
- Apply CPU, memory, filesystem, network, and process limits.
- Disable network access.
- Do not mount train, validation, test, workspace scaffolding, broker endpoints, registry credentials, or any persistent writable directory during private evaluation.

Used by:

- Candidate Evaluation.
- Release Runtime.

### `run_healthcheck`

Input:

- `worker: WorkerHandle`
- `timeout_ms: int`

Output:

- `HealthcheckResult`

Purpose:

- Confirm the worker starts and can answer a healthcheck.
- Fail fast before evaluation or serving if the worker cannot be safely called.

Used by:

- Candidate Evaluation preflight.
- Release warmup.

### `build_worker_request`

Input:

- `request_id: str`
- `input_value: ValidatedInput`
- `deadline_ms: int`
- `request_id_policy: Literal["runtime_stable", "private_eval_ephemeral"]`

Output:

- `WorkerRequest`

Purpose:

- Serialize the target input into the worker protocol without exposing split information, expected output, or evaluator state.
- In private evaluation, generate an ephemeral non-reusable request id that cannot be joined back to source data or evaluation order.

Used by:

- Candidate Evaluation.
- Release Runtime.

### `call_worker`

Input:

- `worker: WorkerHandle`
- `request: WorkerRequest`
- `call_timeout_ms: int`

Output:

- `RawWorkerCallResult`

Purpose:

- Send one request to a running worker.
- Capture stdout/stderr limits, response bytes, latency, timeout, and process failure.
- Does not validate target output correctness.

Used by:

- `run_layer_attempt`

### `parse_worker_response`

Input:

- `raw_result: RawWorkerCallResult`

Output:

- `WorkerResponse | ProtocolError`

Purpose:

- Parse JSONL or socket response.
- Enforce exactly one `accept` or `abstain` decision.
- Reject malformed, oversized, multiple, or missing responses.

Used by:

- `run_layer_attempt`

### `validate_accept_output`

Input:

- `contract: TargetRuntimeContract`
- `response: WorkerResponse`

Output:

- `ValidatedOutput | OutputValidationError`

Purpose:

- Validate the output for `accept` responses.
- Treat invalid output as a layer failure that falls back safely.

Used by:

- `run_layer_attempt`

### `run_layer_attempt`

Input:

- `worker: WorkerHandle`
- `contract: TargetRuntimeContract`
- `request: WorkerRequest`
- `layer: Literal["L1", "L2", "L3"]`

Output:

- `LayerAttemptResult`

Purpose:

- Execute one artifact attempt.
- Validate non-null worker `reason` as a reason code and enforce the artifact manifest allowlist when present.
- Convert abstain, timeout, crash, protocol error, and invalid output into structured attempt results.
- Never call L4 or the next layer.

Used by:

- Candidate Evaluation residual/cascade evaluation.
- Release Runtime request routing.

### `stop_worker`

Input:

- `worker: WorkerHandle`
- `reason: str`

Output:

- `WorkerStopResult`

Purpose:

- Stop a worker and all child processes.
- Record shutdown errors without affecting already completed attempt results.

Used by:

- Candidate Evaluation.
- Release Runtime deployment changes.

### `build_protocol_docs`

Input:

- `protocol_version: str`

Output:

- `ProtocolDocs`

Purpose:

- Export the worker request/response contract and package manifest requirements for the agent workspace.
- This is documentation, not an evaluation loophole.

Used by:

- Agent Workspace module.

## Invariants

- L1/L2/L3 use one worker protocol.
- Artifacts cannot call L4 or each other.
- Runtime artifacts have no network or broker access.
- Private evaluation workers receive ephemeral request ids, randomized request order, and no persistent writable storage.
- Invalid accept output is not returned to users; Core falls back.
- The deployed package contains runtime files only.
- The package bytes evaluated by Core are the package bytes eligible for Release.
- Cache behavior is not implemented inside artifacts.

## Alignment Against 0626-2

- Collapses L1/L2/L3 into resource layers under one runtime ABI.
- Keeps fallback in Core.
- Prevents scaffolding, training data, transcripts, or validation/test data from entering runtime packages.
- Avoids a per-layer compiler class or framework.
