# Security

Darjeeling is an alpha runtime. It is ready for design validation and early demo
feedback, not production deployment.

## Supported Versions

| Version | Supported |
| --- | --- |
| `0.1.0-alpha.1` | Security reports accepted |

## Report A Vulnerability

Use GitHub's private vulnerability reporting for this repository if it is
enabled. If it is not enabled, contact the maintainer through the repository
profile and ask for a private reporting channel.

Do not open a public issue that includes exploit details, secrets, private
target data, or provider credentials.

## Scope

Security reports are most useful when they cover:

- unsafe artifact packaging or execution behavior;
- leakage of raw target data into traces, reports, or agent-visible files;
- bypasses of target contract validation;
- unsafe handling of API keys or provider errors;
- repository supply-chain risks.

Demo feedback and general feature requests should use the normal issue forms.
