# 2026-06-08 Experiment Plan

Goal: validate the proposal thesis with L0/L1/L2/L3/L4 all enabled in the main
run, then compare required ablations at controlled scale.

## Fixed Configuration

- L4 model: `gpt-5.5`
- Prompt cache retention: `24h`. The current GPT 5.5 backend rejects
  `prompt_cache_retention=in_memory`.
- Teacher mode: `live-or-cache`
- L4 proposal mode: `live`
- L1 agent mode: `codex-cli`
- L1 agent model: `gpt-5.5`
- L3 mode: `shadow` for the main run, so the real local SLM runs and records
  quality/latency without making the full experiment brittle on local hardware.
- L3 model: `Qwen/Qwen2.5-0.5B-Instruct`
- Retry: `OPENAI_MAX_RETRIES=5`, exponential backoff starting at 2 seconds.
- Timeout: `OPENAI_TIMEOUT_S=20`, so a stalled request retries instead of
  hanging a whole compiler iteration.
- Parallelism: run independent experiment subprocesses with `edge-mvp-nlu experiment
  suite --parallel 2`; reduce to 1 if local SLM memory pressure appears.

## Budget

Hard budget: $100.

Plan:

1. Pilot stays below 100 requests and 3 compiler iterations.
2. Inspect `teacher_cache.jsonl`, `reports/metrics.csv`, and actual L4 usage
   after pilot.
3. Full suite scale is capped by observed Codex-agent cost/time. The r8 pilot
   showed 60 requests with `compile_every=20` triggers 3 real Codex agent jobs,
   so follow-up ablations use at most one compiler iteration per experiment.
4. Stop immediately if projected total L4/API spend exceeds $85; keep $15
   reserve for reruns after fixes.

The report cost model is treated as a run-local estimate. Actual provider
billing remains authoritative, so scale only grows after pilot usage is known.

## Stages

### Stage 0: Readiness

Commands:

```bash
uv sync --extra dev
uv run edge-mvp prepare --locale en-US
OPENAI_MODEL=gpt-5.5 \
PROMPT_CACHE_RETENTION=24h \
L4_PROPOSAL_MODE=live \
L1_AGENT_MODE=codex-cli \
L1_AGENT_MODEL=gpt-5.5 \
LOCAL_SLM_MODE=shadow \
OPENAI_MAX_RETRIES=5 \
OPENAI_RETRY_BASE_DELAY_S=2 \
OPENAI_TIMEOUT_S=20 \
uv run edge-mvp-nlu experiment preflight --run-dir runs/exp-full/main --teacher live-or-cache --check-l1-build
```

Run L3 hardware preflight:

```bash
LOCAL_SLM_MODE=shadow \
PROMPT_CACHE_RETENTION=24h \
uv run edge-mvp-nlu l3 bench --out runs/exp-full/main/reports/l3_benchmark.json
```

### Stage 1: Full-Layer Pilot

```bash
OPENAI_MODEL=gpt-5.5 \
PROMPT_CACHE_RETENTION=24h \
L4_PROPOSAL_MODE=live \
L1_AGENT_MODE=codex-cli \
L1_AGENT_MODEL=gpt-5.5 \
LOCAL_SLM_MODE=shadow \
OPENAI_MAX_RETRIES=5 \
OPENAI_RETRY_BASE_DELAY_S=2 \
OPENAI_TIMEOUT_S=20 \
uv run edge-mvp-nlu experiment main-evolution \
  --run-dir runs/exp-full/main-pilot \
  --max-requests 60 \
  --compile-every 20 \
  --teacher live-or-cache
```

Acceptance for moving on:

- Run completes and writes `traces.jsonl`.
- `summary.md`, `metrics.csv`, `curves.html`, and `hard_cases.jsonl` exist.
- L4 live calls are recorded or cache hits are traceable.
- L1 agent artifacts are present for at least one generation or a concrete agent
  failure is recorded in candidate metrics.
- L3 shadow results are present, or report clearly records hardware/model
  failure and degraded mode.

### Stage 2: Controlled Suite

Run the required experiments with shared settings at reduced scale:

```bash
OPENAI_MODEL=gpt-5.5 \
PROMPT_CACHE_RETENTION=24h \
L4_PROPOSAL_MODE=live \
L1_AGENT_MODE=codex-cli \
L1_AGENT_MODEL=gpt-5.5 \
LOCAL_SLM_MODE=shadow \
OPENAI_MAX_RETRIES=5 \
OPENAI_RETRY_BASE_DELAY_S=2 \
OPENAI_TIMEOUT_S=20 \
uv run edge-mvp-nlu experiment suite \
  --run-root runs/exp-full/suite \
  --max-requests 40 \
  --compile-every 40 \
  --parallel 1 \
  --teacher live-or-cache
```

Rationale: the main full-layer pilot already validates live L4 fallback, live L4
proposal, real Codex CLI L1 evolution, L2 training/guarding, L0 cache promotion,
and L3 shadow execution. The controlled suite is therefore used to compare
directions under budget rather than to produce statistically stable estimates.
Use `parallel=1` because local L3 uses MPS and each compiler iteration may launch
a long-running Codex agent process.

Suite includes:

- `main-evolution`
- `direct-l4-optimization`
- `l2-family`
- `no-guard`
- `no-l2`
- `hard-buffer`
- `workload-locality`

### Stage 3: Analysis

Generate or verify:

```bash
uv run edge-mvp-nlu experiment compare --root runs/exp-full --out-dir runs/exp-full/comparison
```

Read:

- layer summary
- evolution summary
- artifact summary
- L1 readable program/diff section
- L3 hardware and shadow stats
- failed experiment analysis

## Repair Loop

If any stage fails:

1. Record the failing command and log path.
2. Return to development.
3. Add or update tests when the failure is code-level.
4. Commit the fix.
5. Resume from the smallest stage that proves the fix.

Every code/config/doc change during the experiment must be committed before the
next experimental run.
